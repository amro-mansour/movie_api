Documentation Page

The objective of this api is to build the server side component of a movies web application. The web application will provide users information about different movies, actors, genres, directors, etc. Users will also be allowed to sign up, add and update their personal information, and create a list of their favourite movies.

These are the instructions on how to use the API

Logic URL HTTP Method Query Parameters Request Data Body Format Response Data Body Format
Return a list of ALL movies to the user /movies GET None None JSON data type containing details about all the movies
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/:Title GET :Title None JSON object containing details of specific movie, e.g.

movie = {
Title: "",
Description: "",
Genre: {
Name: "",
Description: ""
},
Director: {
Name: "",
Bio: "",
Birth: ""
},
Actors: [],
ImagePath: "",
}
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/genre/:genreName GET :genreName None Json object containg information about a specific movie genre, e.g.

Genre: {
Name: "Crime",
Description: ""
},
Return data about a director (bio, birth year, death year) by name /movies/directors/:directorName GET :directorName None JSON object containing information about a specific director (bio, birth year, death year), e.g.

Director: {
Name: "Guy Ritchie",
Bio: "",
Birth: ""
}
Allow new users to register /users POST None JSON object is added, e.g.

{
"Username": "william.smith_87",
"Password": "kl292qw9",
"Email": "william_smith87@gmail.co.uk",
"Birthday": "1987-02-12",
"FavouriteMovies": [ ],
} 		JSON object sent back from the request plus a random ID number assigned to the user, e.g. {
"_id": "625d456453a8f15782ee5122" "Username": "william.smith_87",
"Password": "kl292qw9",
"Email": "william_smith87@gmail.co.uk",
"Birthday": "1987-02-12",
"FavouriteMovies": [ ],
}
Allow users to update their user info (username) /users/:Username PUT :Username JSON object containing the data to modify, e.g. {"name": "New Name"} Json object sent back from the request with updated infromation, e.g.
{"name": "New Name"}
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added) /users/:Username/movies/:MovieID POST :Username, :MovieID None A message reporting the movie has been added succefully
Allow users to remove a movie from their list of favorites /users/:Username/movies/:MovieID DELETE :Username, :MovieID None A message reporting that the movie has been succefully deleted
Allow existing users to deregister /users/:Username DELETE :Username None A message reporting that the user has been succefully deleted
